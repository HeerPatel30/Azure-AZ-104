# 1. What is Self-Service Password Reset (SSPR)?

**Microsoft Entra Self-Service Password Reset (SSPR)** allows users to securely reset or unlock their own passwords without contacting the IT help desk.

Users verify their identity using one or more configured authentication methods before resetting their password.

SSPR helps organizations:

- Reduce help desk calls
- Improve user productivity
- Strengthen security through identity verification
- Enable users to regain access anytime

---

## Example

An employee forgets their Microsoft 365 password while working remotely.

Instead of contacting IT support, they:

1. Verify their identity.
2. Create a new password.
3. Sign in successfully.

---

# 2. Why Use SSPR?

Without SSPR:

- Users must contact the IT help desk.
- Password resets consume IT resources.
- Productivity decreases.
- Remote users may experience delays.

SSPR provides:

- Faster password recovery
- Reduced support costs
- Improved security
- Better user experience
- 24/7 password reset capability

---

# 3. How SSPR Works

The SSPR process follows these steps:

```
User Forgets Password
          │
          ▼
Clicks "Forgot Password"
          │
          ▼
Microsoft Entra ID
Verifies Identity
          │
          ▼
Authentication Successful?
      │             │
     Yes            No
      │             │
      ▼             ▼
Create New      Password
Password        Reset Denied
      │
      ▼
Password Updated
      │
      ▼
User Signs In
```

# 4. Prerequisites

Before enabling SSPR, ensure the following requirements are met:

- Microsoft Entra tenant
- Appropriate Microsoft Entra license
- Administrator permissions
- Registered authentication methods
- Internet access

For hybrid environments:

- Microsoft Entra Connect
- Password Writeback enabled

# 6. Enable Self-Service Password Reset

Administrators enable SSPR in the Microsoft Entra admin center.

Navigation:

```
Microsoft Entra Admin Center

↓

Protection

↓

Password Reset

↓

Properties
```

Administrators choose who can use SSPR:

- None
- Selected Group(s)
- All Users

---

## Recommended Deployment

Microsoft recommends:

1. Enable SSPR for a pilot group.
2. Verify functionality.
3. Expand to additional groups.
4. Enable for all users.

# 7. Authentication Methods

Before resetting a password, users must verify their identity.

Microsoft Entra supports multiple authentication methods.

---

## Mobile App Notification

User approves a notification from Microsoft Authenticator.

---

## Mobile App Verification Code

User enters a code generated by Microsoft Authenticator.

---

## Mobile Phone

Verification by:

- SMS
- Voice call

---

## Office Phone

Verification through an office telephone.

---

## Email

A verification code is sent to a registered email address.

> **Note:** Email can be used only for password reset and isn't available for administrator accounts.

---

## Security Questions

Users answer preconfigured security questions.

Common examples:

- First school attended
- Favorite teacher
- First pet's name

---

## Authentication Methods Summary

| Method | Available for Standard Users | Available for Administrators |
|---------|:---------------------------:|:----------------------------:|

| Voice Call | ✅ | ✅ |
| Office Phone | ✅ | ❌ |
| Email | ✅ | ❌ |
| Security Questions | ✅ | ❌ |

# 8. Authentication Method Requirements

Administrators configure how many authentication methods are required.

Two options are available:

### One Method Required

Example:

- SMS only

---

### Two Methods Required

Example:

- SMS
- Microsoft Authenticator

Using two methods provides stronger security.

# 9. Password Writeback

Password Writeback allows password changes made in Microsoft Entra ID to be written back to the on-premises Active Directory.

This is required in hybrid environments.

---

## Workflow

```
User Resets Password
          │
          ▼
Microsoft Entra ID
          │
          ▼
Microsoft Entra Connect
          │
          ▼
On-Premises Active Directory
Password Updated
```

---

## Benefits

- Single password everywhere
- Hybrid identity support
- Improved user experience
- Reduced administrative effort

---

# 10. User Registration

Before using SSPR, users must register authentication methods.

Typical registration includes:

- Mobile phone
- Microsoft Authenticator
- Email
- Security questions

Without registration, password reset cannot be completed.

---

# 11. Combined Security Information Registration

Microsoft recommends enabling **Combined Security Information Registration**.

This allows users to register:

- Multi-Factor Authentication (MFA)
- Self-Service Password Reset (SSPR)

using a **single registration experience**.

Benefits include:

- Simpler user experience
- Reduced registration time
- Easier administration
- Improved security

---

# 12. Notifications

Microsoft Entra can send notifications after password events.

Notifications include:

- Password reset confirmation
- Password change confirmation
- Administrator alerts

Benefits:

- Detect unauthorized password changes
- Improve security monitoring
- Inform users immediately

# 13. Reporting and Auditing

Administrators can review SSPR activity.

These reports help identify:

- Security issues

- Troubleshooting information

---

# 14. Password Reset vs Password Change

These terms are commonly confused.

| Password Change | Password Reset |
|----------------|---------------|
| User knows current password | User forgot password |
| Current password required | Identity verification required |
| User is signed in | User is not signed in |
| Password updated | Password recovered |

---

# 15. Password Policies

When resetting passwords, organizations can enforce password policies.

Typical requirements include:

- Minimum password length
- Password complexity
- Password history
- Password expiration (if applicable)
- Restricted common passwords

Strong password policies improve overall security.

---

# 16. Best Practices

Microsoft recommends:

- Enable SSPR for all eligible users.
- Require at least two authentication methods.
- Enable Combined Security Information Registration.
- Enable Password Writeback for hybrid environments.
- Encourage Microsoft Authenticator over SMS.
- Monitor SSPR reports regularly.
- Educate users about password security.
- Pilot SSPR before organization-wide deployment.

---

# 18. AZ-104 Exam Tips

Remember these important concepts:

- SSPR allows users to reset their own passwords.
- Identity verification is required before resetting a password.
- Password Writeback requires Microsoft Entra Connect.
- Password Writeback is supported with Microsoft Entra ID P1 or P2 licensing.
- Combined Registration configures MFA and SSPR together.
- Microsoft Authenticator is the recommended authentication method.
- Password Reset and Password Change are different operations.
- Pilot SSPR before enabling it for all users.
- Password reset reports are available in Microsoft Entra.

# 20. Summary

Microsoft Entra Self-Service Password Reset (SSPR) enables users to securely reset or unlock their passwords without contacting the IT help desk. By verifying identity through configured authentication methods, users can quickly regain access to their accounts while reducing support costs and improving productivity.

For hybrid environments, Password Writeback ensures password changes are synchronized to on-premises Active Directory. Combined Security Information Registration simplifies the setup of both Multi-Factor Authentication (MFA) and SSPR, providing a seamless user experience. When combined with strong password policies and Microsoft Authenticator, SSPR becomes an essential part of a secure identity management strategy.
